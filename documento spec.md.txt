# 🩺 Help Alpha — Sistema de Consulta de Exames

## 📘 Visão Geral

O **Help Alpha** é um sistema web desenvolvido com **Flask (Python)** e **Bulma (CSS)** com o objetivo de oferecer aos pacientes uma plataforma simples, responsiva e profissional para consulta de informações sobre exames laboratoriais e de imagem.

---

## 🎯 Objetivo do Projeto

- Facilitar o acesso dos pacientes às informações sobre exames.
- Reduzir dúvidas recorrentes no atendimento.
- Centralizar orientações de preparo, documentos exigidos e cuidados.
- Tornar a experiência do usuário intuitiva e acessível em qualquer dispositivo.
- Fornecer informações sobre unidades, horários e contatos.
- Permitir gerenciamento eficiente de exames via interface administrativa.
- Suportar importação em massa de dados via Excel.

---

## 🧱 Tecnologias Utilizadas

| Camada       | Tecnologia                    |
|--------------|-------------------------------|
| Backend      | Python + Flask               |
| Frontend     | Bulma CSS Framework          |
| Autenticação | Flask-Login                  |
| Formulários  | Flask-WTF + WTForms          |
| Dados        | JSON (estrutura escalável)   |
| Excel        | pandas + openpyxl + xlrd     |
| Hospedagem   | Compatível com Render, Railway, etc |

## 📑 Estrutura dos Dados

### Formato JSON
```json
[
  {
    "nome": "Hemograma Completo",
    "descricao": "Avaliação das células sanguíneas.",
    "preparo": "Jejum de 8 horas.",
    "documentos": "RG e pedido médico.",
    "pos_exame": "Nenhuma restrição.",
    "tempo": "10 minutos"
  }
]
```

### Formato Excel para Upload
| Coluna | Descrição | Obrigatório |
|--------|-----------|-------------|
| nome | Nome do exame | ✅ |
| descricao | Descrição detalhada | ✅ |
| preparo | Instruções de preparo | ✅ |
| documentos | Documentos necessários | ❌ |
| pos_exame | Cuidados pós-exame | ❌ |
| tempo | Duração estimada | ❌ |

## 🧠 Regras de Negócio

* Todos os exames devem conter, no mínimo: nome, descrição e preparo.
* A busca deve retornar resultados parciais com base no nome digitado.
* O sistema deve ser funcional e responsivo em qualquer tipo de dispositivo.
* Deve manter separação entre dados e interface.
* Informações de unidades devem estar sempre visíveis e atualizadas.
* Upload Excel deve validar campos obrigatórios automaticamente.
* Dados vazios no Excel devem ser preenchidos com valores padrão.
* Sistema deve prevenir duplicatas por nome de exame.
